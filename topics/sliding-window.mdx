The sliding window technique maintains a contiguous subarray or substring defined by two pointers (`i` and `j`). By expanding the right boundary and shrinking the left boundary under a condition, you visit each element at most twice — turning O(n²) brute force into O(n).

The repository contains **55+ sliding window solutions** covering both fixed-size and variable-size variants.

<Note>
  Sliding window only applies to problems where the answer is a **contiguous** subarray or substring. If the problem asks for subsequences (elements need not be adjacent), use DP or two pointers instead.
</Note>

## Two Window Patterns

<Tabs>
  <Tab title="Fixed-size window (size k)">
    The window always has exactly `k` elements. Slide it one step at a time: add the new right element, remove the old left element.

    **When to use:** "find max/min/sum/count in every subarray of size k"

    ```cpp
    // Template: fixed window of size k
    int windowSum = 0;
    for (int i = 0; i < k; i++)
        windowSum += nums[i];          // build initial window

    int result = windowSum;
    for (int i = k; i < nums.size(); i++) {
        windowSum += nums[i];          // expand right
        windowSum -= nums[i - k];     // shrink left (drop outgoing element)
        result = max(result, windowSum);
    }
    ```

    **Problems in this pattern:**
    - Maximum Sum of Distinct Subarrays With Length K
    - Maximum Number of Vowels in a Substring of Given Length
    - First Negative Integer in Every Window of Size k
    - Grumpy Bookstore Owner
    - K Radius Subarray Averages
    - Find the Power of K-Size Subarrays I
    - Sliding Window Maximum (use deque for O(1) max)
  </Tab>
  <Tab title="Variable-size window (shrinkable)">
    The window grows right until a condition is violated, then shrinks from the left until it is valid again.

    **When to use:** "find shortest/longest subarray/substring satisfying condition X"

    ```cpp
    // Template: variable window
    int i = 0;
    // maintain window state (e.g. a frequency map)
    for (int j = 0; j < n; j++) {
        // expand: include s[j] in window state
        updateState(s[j]);

        while (windowViolatesCondition()) {
            // shrink: remove s[i] from window state
            removeFromState(s[i]);
            i++;
        }

        // window [i..j] is now valid — update answer
        result = max(result, j - i + 1);
    }
    ```

    **Problems in this pattern:**
    - Minimum Window Substring
    - Longest Substring Without Repeating Characters
    - Longest K Unique Characters Substring
    - Fruit Into Baskets
    - Minimum Size Subarray Sum
    - Subarray Product Less Than K
    - Frequency of the Most Frequent Element
  </Tab>
</Tabs>

## Code: Minimum Window Substring

**Company Tags:** Google, Amazon, Microsoft, Codenation, FactSet, Atlassian, MakeMyTrip, Airbnb

**T.C:** O(m + n) where m = `|s|`, n = `|t|`
**S.C:** O(n) for the frequency map

```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        int n = s.length();
        map<char, int> mp;

        // Record required character frequencies from t
        for (char& ch : t)
            mp[ch]++;

        int requiredCount = t.length(); // characters still needed
        int i = 0, j = 0;
        int minStart  = 0;
        int minWindow = INT_MAX;

        while (j < n) {
            char ch_j = s[j];

            // Expanding right: if this char was needed, satisfy one requirement
            if (mp[ch_j] > 0)
                requiredCount--;
            mp[ch_j]--;

            // All required chars are in the window — try to shrink from left
            while (requiredCount == 0) {
                if (minWindow > j - i + 1) {
                    minWindow = j - i + 1;
                    minStart  = i;
                }

                char ch_i = s[i];
                mp[ch_i]++;
                // If removing s[i] breaks coverage, increment requiredCount
                if (mp[ch_i] > 0)
                    requiredCount++;
                i++;
            }

            j++;
        }

        return minWindow == INT_MAX ? "" : s.substr(minStart, minWindow);
    }
};
```

## When to Shrink the Window

<Tip>
  Shrink the left pointer when the window **violates your constraint**. For minimization problems (shortest window), shrink as soon as the constraint is satisfied. For maximization problems (longest window), shrink when the constraint is violated.
</Tip>

| Goal | Shrink when |
|------|-------------|
| Shortest valid window | Constraint is satisfied — shrink to minimize length |
| Longest valid window | Constraint is violated — shrink to restore validity |
| Count valid windows | Depends on whether condition is "at most" or "exactly" |

<Note>
  **Exactly-k trick:** `count(exactly k) = count(at most k) − count(at most k−1)`. This converts an "exactly k" problem into two "at most k" sliding window runs.
</Note>
