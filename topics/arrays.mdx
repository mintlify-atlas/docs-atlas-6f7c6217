Arrays are the foundation of nearly every coding interview. They appear in problems spanning prefix sums, sorting, searching, and complex simulation. Mastering the core patterns here unlocks solutions across linked lists, strings, and dynamic programming as well.

## Technique Patterns

### HashMap-based O(n) Search — Two Sum

The classic "complement lookup" pattern: store each element in a hashmap as you scan, and check whether the complement of the current element has already been seen.

```cpp
// T.C : O(n)   S.C : O(n)
// Company Tags: Zoho, Flipkart, Amazon, Microsoft, Google, Adobe
vector<int> twoSum(vector<int>& nums, int target) {
    unordered_map<int, int> mp;

    for (int i = 0; i < nums.size(); i++) {
        if (mp.count(target - nums[i]))
            return {mp[target - nums[i]], i};
        mp[nums[i]] = i;
    }

    return {};
}
```

### Kadane's Algorithm — Maximum Subarray

The key insight: at each position, either extend the current subarray or start fresh from the current element — whichever is larger.

```cpp
// T.C : O(n)   S.C : O(1)
// Company Tags: MWare, Zoho, Amazon, Microsoft, Oracle, Visa, Walmart
int maxSubArray(vector<int>& nums) {
    int sum    = nums[0];
    int maxSum = nums[0];

    for (int i = 1; i < nums.size(); i++) {
        sum    = max(sum + nums[i], nums[i]);
        maxSum = max(maxSum, sum);
    }

    return maxSum;
}
```

### Prefix/Suffix Product — Product of Array Except Self

Build a left-product array in one pass and accumulate the right product in a second pass — no division needed.

```cpp
// T.C : O(n)   S.C : O(1)  (output array not counted)
// Company Tags: Apple, LinkedIn, Amazon, Facebook, Microsoft
vector<int> productExceptSelf(vector<int>& nums) {
    int n = nums.size();
    vector<int> result(n);
    result[0] = 1;

    for (int i = 1; i < n; i++)
        result[i] = result[i - 1] * nums[i - 1];

    int right = 1;
    for (int i = n - 1; i >= 0; i--) {
        result[i] *= right;
        right     *= nums[i];
    }

    return result;
}
```

## Tips for Interviews

<Tip>
When asked for a subarray sum, prefix sums are almost always the right first move. Build a `prefixSum` array and use a hashmap to find if `prefixSum[j] - prefixSum[i] == target` in O(n).
</Tip>

<Tip>
Two-pointer problems require the array to be sorted (or have a monotonic property). Always ask yourself: "can I sort first without violating constraints?" before reaching for O(n²) brute force.
</Tip>

<Tip>
For "maximum subarray" variants (circular, with length divisible by K, etc.), always start from plain Kadane's and think about what boundary condition changes. The core `max(sum + nums[i], nums[i])` logic rarely changes.
</Tip>

<Tip>
Difference arrays shine when you have many range-update queries followed by a single read pass. If you see "add value V to every index from L to R" repeated many times, reach for `diff[L] += V; diff[R+1] -= V;`.
</Tip>

<Tip>
For 2-D grid problems, always check whether a 2-D prefix sum (summed over rows and columns) reduces the problem to O(1) per query from O(m*n).
</Tip>
