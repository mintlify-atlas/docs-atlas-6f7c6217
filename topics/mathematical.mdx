## Sub-Topics

<AccordionGroup>
  <Accordion title="Number Theory — GCD, LCM, Prime Sieve">
    **GCD / LCM** appear in problems involving fractions, rotations, and cycle lengths.

    ```cpp
    // Euclidean algorithm — O(log(min(a,b)))
    int gcd(int a, int b) {
        return b == 0 ? a : gcd(b, a % b);
    }
    int lcm(int a, int b) {
        return a / gcd(a, b) * b; // divide first to avoid overflow
    }
    ```

    **Sieve of Eratosthenes** generates all primes up to N in O(N log log N):

    ```cpp
    vector<bool> sieve(int n) {
        vector<bool> isPrime(n + 1, true);
        isPrime[0] = isPrime[1] = false;
        for (int i = 2; (long long)i * i <= n; i++) {
            if (isPrime[i]) {
                for (int j = i * i; j <= n; j += i)
                    isPrime[j] = false;
            }
        }
        return isPrime;
    }
    ```

    **Problems:** Closest Prime Numbers in Range, Segmented Sieve, Four Divisors, Ugly Number
  </Accordion>

  <Accordion title="Combinatorics and Counting">
    Many counting problems reduce to **nCr mod p** or a closed-form product.

    ```cpp
    // Factorials with modular arithmetic
    const int MOD = 1e9 + 7;
    long long fact[100001];
    void precompute() {
        fact[0] = 1;
        for (int i = 1; i <= 100000; i++)
            fact[i] = fact[i - 1] * i % MOD;
    }
    // nCr mod p
    long long power(long long base, long long exp, long long mod) {
        long long result = 1;
        while (exp > 0) {
            if (exp & 1) result = result * base % mod;
            base = base * base % mod;
            exp >>= 1;
        }
        return result;
    }
    long long nCr(int n, int r) {
        return fact[n] * power(fact[r], MOD - 2, MOD) % MOD
                       * power(fact[n - r], MOD - 2, MOD) % MOD;
    }
    ```

    **Problems:** Count All Valid Pickup and Delivery Options, New 21 Game, Number of Ways to Reorder Array to Get Same BST
  </Accordion>

  <Accordion title="Geometry">
    Geometry problems involve points, lines, slopes, and areas. Watch for floating-point precision — prefer integer arithmetic with cross products when possible.

    ```cpp
    // Cross product to determine orientation of three points
    long long cross(pair<int,int> O, pair<int,int> A, pair<int,int> B) {
        return (long long)(A.first - O.first)  * (B.second - O.second)
             - (long long)(A.second - O.second) * (B.first - O.first);
    }
    ```

    **Problems:** Max Points on a Line, Erect the Fence (Convex Hull), Generate Random Point in a Circle, Minimum Time Visiting All Points
  </Accordion>

  <Accordion title="Modular Arithmetic">
    Any problem with large answers asks for the result **mod 10^9+7**. Key rules:

    - `(a + b) % m = ((a % m) + (b % m)) % m`
    - `(a * b) % m = ((a % m) * (b % m)) % m`
    - Modular inverse via Fermat's little theorem: `a^(m-2) mod m` (when m is prime)
    - **Never** take `%` after subtraction without adding `m` first to avoid negative results

    ```cpp
    long long modInverse(long long a, long long m) {
        return power(a, m - 2, m); // m must be prime
    }
    ```
  </Accordion>

  <Accordion title="Matrix Exponentiation">
    Compute the Nth term of a linear recurrence (Fibonacci, etc.) in **O(k³ log N)** by raising the transformation matrix to the Nth power.

    ```cpp
    // Matrix multiply (for 2x2 Fibonacci matrix)
    typedef vector<vector<long long>> Matrix;
    Matrix multiply(const Matrix& A, const Matrix& B, long long mod) {
        int n = A.size();
        Matrix C(n, vector<long long>(n, 0));
        for (int i = 0; i < n; i++)
            for (int k = 0; k < n; k++)
                for (int j = 0; j < n; j++)
                    C[i][j] = (C[i][j] + A[i][k] * B[k][j]) % mod;
        return C;
    }
    ```

    **Problems:** Matrix Exponentiation (repo), Fibonacci in O(log n), Climbing Stairs variants
  </Accordion>
</AccordionGroup>

## Worked Example: Pow(x, n) — Fast Exponentiation

> **Company Tags:** Amazon, LinkedIn, Bloomberg

Naive `x^n` via repeated multiplication is O(n). Fast exponentiation halves the exponent each step, giving **O(log n)**.

**Key insight:** `x^n = (x²)^(n/2)` when n is even; `x^n = x · (x²)^((n-1)/2)` when n is odd.

```cpp
// Company Tags: Amazon, LinkedIn, Bloomberg
// Leetcode: https://leetcode.com/problems/powx-n/

class Solution {
public:
    double solve(double x, long n) {
        if (n == 0) return 1;          // base case
        if (n < 0)  return 1 / solve(x, -n); // handle negative exponent

        if (n % 2 == 0) {
            return solve(x * x, n / 2);       // even: square and halve
        }
        return x * solve(x * x, (n - 1) / 2); // odd: factor out one x
    }

    double myPow(double x, int n) {
        return solve(x, (long)n); // cast to long: n can be INT_MIN
    }
};
```

<Note>
  The cast to `long` is critical. If `n = INT_MIN` (-2147483648), multiplying by -1 overflows a 32-bit integer. Using `long` avoids this edge case.
</Note>

## Useful Formulas to Memorize

| Formula | Use case |
|---------|----------|
| `1 + 2 + ... + n = n*(n+1)/2` | Sum of first N integers |
| `1² + 2² + ... + n² = n*(n+1)*(2n+1)/6` | Sum of squares |
| `nCr = n! / (r! * (n-r)!)` | Combinations |
| `a^b mod p` via fast exp | Modular exponentiation |
| `gcd(a, b) = gcd(b, a%b)` | GCD — Euclidean algorithm |

<Tip>
  When a problem asks "how many ways" or "count the arrangements," think combinatorics first. Sum of 1..n and related closed forms often convert an O(n) loop into an O(1) formula.
</Tip>
